# 微處理機應用-Arduino

### `Buzzer_Melody.ino` (蜂鳴器音樂播放)
* **功能概述**：透過字元與陣列映射，控制蜂鳴器依序播放自訂旋律（範例為「小星星」），並可彈性調整節奏與拍長。
* **實作細節與硬體邏輯**：
  * **雙陣列映射機制**：利用字元陣列 `notes[]` 紀錄簡譜音符，配合 `beats[]` 陣列記錄節拍長度。在自訂函式 `playNote()` 中，進一步將字元（如 'c', 'd', 'e'）對應至實際的音訊頻率 (261Hz, 294Hz 等)。
  * **訊號輸出與時序控制 (Timing Control)**：
    * **防止音符重疊**：Arduino 的 `tone()` 函式在送出 PWM 頻率後會馬上執行下一行指令。程式碼中特別在 `tone()` 後方加入 `delay(duration)` 進行阻擋 (Blocking)，確保音符完整播放後再呼叫 `noTone()` 停止發聲，避免音訊混雜。
    * **連音與斷音效果**：主迴圈 `loop()` 中，在每個音符結束後加入了 `delay(tempo/10)` 的微小間隔，讓連續的音符之間有明確的段落感。
  * **硬體腳位**：預設使用 Pin 18 作為 PWM 訊號輸出腳位（符合 ESP32 常用之 GPIO 設定），遇到空白字元 `' '` 則視為休止符，僅作延遲而不輸出頻率。
 
    ### `Keypad_LCD_Password.ino` (矩陣鍵盤與 LCD 密碼鎖系統)
* **功能概述**：整合 4x4 薄膜矩陣鍵盤與 1602 LCD 顯示器，實作一組具備輸入遮罩（顯示 `*`）、退格刪除（按 `D` 鍵）與自動驗證功能的密碼鎖系統。
* **實作細節與硬體邏輯**：
  * **底層矩陣掃描機制 (Matrix Scanning)**：不依賴外部 Keypad 函式庫，自訂 `scan()` 函式進行硬體輪詢。透過將 Row 腳位依序設為 `LOW`，並讀取啟用內建上拉電阻 (`INPUT_PULLUP`) 的 Column 腳位狀態，精準抓出按鍵座標 `(x, y)` 並對應到 `tbl` 字元陣列。
  * **狀態機設計與非阻塞延遲 (Non-blocking Delay)**：
    * 系統架構分為 `WAITING_INPUT` (等待輸入) 與 `SHOW_RESULT` (顯示驗證結果) 兩種狀態，避免輸入過程產生邏輯衝突。
    * 在顯示密碼正確或錯誤畫面時，捨棄了會使系統停擺的 `delay()`，改為使用 `millis() - lastStateChangeTime > displayDelay` 進行非阻塞式計時。這確保了系統在維持畫面 5 秒後，能順暢地呼叫 `resetSystem()` 回歸初始狀態。
  * **邊緣觸發 (Edge Triggering) 防呆機制**：利用 `now == 1 && old == 0` 判斷按鍵狀態的下降沿（從未按到按下）。這個技巧等同於實作了防彈跳與單次觸發邏輯，能有效防止長按時連續輸入重複字元。
  * **硬體腳位特性**：程式中使用的 GPIO 腳位（如 27, 26, 33, 32）完美對應 ESP32 的硬體環境，展示了將基礎 I/O 元件整合至 ESP32 系統的實作能力。

### `Keypad_7Segment_Diag.ino` (矩陣鍵盤連動七段顯示器與硬體診斷)
* **功能概述**：透過 `Keypad.h` 函式庫讀取 4x4 矩陣鍵盤的輸入，將按下的數字同步顯示於共陰極七段顯示器上。同時，內建定時底層腳位診斷功能，可於序列埠輸出矩陣按鍵的原始接通狀態。
* **實作細節與硬體邏輯**：
  * **字元轉整數的 ASCII 運算**：在讀取到鍵盤字元後，利用 `key - '0'` 的 ASCII 碼位移技巧，將字元型態的 `'0'~'9'` 快速轉換為整數型態的 `0~9`，以此作為二維陣列 `digits` 的索引來控制七段顯示器的 a-g 腳位。
  * **七段顯示器驅動 (共陰極設定)**：宣告了 `digits[10][7]` 的二維陣列來儲存 0-9 的位元遮罩。因為採用共陰極 (Common Cathode) 設計，陣列中的 `1` 代表輸出高電位 (`HIGH`) 使該段 LED 點亮；若按下非數字鍵 (如 A, B, *, #)，則呼叫 `clearDisplay()` 輸出全 `LOW` 關閉顯示。
  * **硬體診斷掃描器 (`diagScan`)**：這是本程式的一大亮點。除了依賴 `Keypad.h` 抓取鍵值，程式實作了手動輪詢機制的底層檢測：
    * 定期將特定 Column 腳位設為輸出並拉低至 `LOW`。
    * 讀取開啟內建上拉電阻 (`INPUT_PULLUP`) 的 Row 腳位。
    * 透過 Serial Monitor 印出類似 `C0->R0=1` 的原始腳位狀態（若按下按鍵，該交點會短路接地，狀態變為 `0`）。這能極大化地幫助開發者排除 ESP32 麵包板的杜邦線接觸不良或接線錯誤問題。
  * **非阻塞定時器**：主迴圈利用 `millis() - lastDiagMillis >= 1000` 實作非阻塞式計時，確保每秒執行一次硬體診斷掃描的同時，完全不會延遲或卡住鍵盤的即時輸入偵測。
### `OLED_Blink_Animation.ino` (OLED 點陣圖動畫與文字顯示)
* **功能概述**：整合 `Adafruit_SSD1306` 與 `Adafruit_GFX` 函式庫，在 128x64 解析度的 I2C OLED 螢幕上實作動態眨眼動畫，並於畫面下方精準定位顯示自訂字串（如學號）。
* **實作細節與硬體邏輯**：

| 核心技術與概念 | 說明與實作對應 |
| :--- | :--- |
| **PROGMEM 記憶體管理** | 128x32 的點陣圖陣列資料量較大，程式中特別使用了 `PROGMEM` 關鍵字將 `image1` 與 `image2` 強制編譯並儲存於微控制器的快閃記憶體 (Flash Memory) 中。這能有效避免耗盡寶貴的 SRAM，是資源受限的嵌入式系統中非常關鍵的最佳化技巧。 |
| **影格緩衝區 (Frame Buffer)** | Adafruit GFX 的繪圖邏輯是先在記憶體中完成作畫。程式透過 `clearDisplay()` 清空前一幀，接著用 `drawBitmap()` 與 `print()` 排版圖文，最後必須呼叫 `display()` 才會將緩衝區的畫面一次性推送到 OLED 螢幕上，確保動畫播放時不會產生撕裂或閃爍。 |
| **座標系統與圖文排版** | 螢幕總解析度為 128x64。動畫影像從左上角 `(0, 0)` 開始繪製，剛好佔滿上半部（高 32 像素）；文字則利用 `setCursor(40, 48)` 將游標移至下半部置中位置，達成圖文互不干擾的完美分割。 |
| **狀態切換與時序** | 在 `loop()` 迴圈中，利用 `delay(500)` 控制每個畫面的停留時間。每半秒切換一次「睜眼」與「閉眼」的點陣圖，藉此形成生動的視覺回饋效果。 |

**總結**：
本程式完美展示了 I2C 通訊協定的硬體周邊控制能力。從初始化位址 (`0x3C`)、記憶體最佳化分配，到透過程式邏輯控制幀率 (FPS) 產生動畫效果，完整涵蓋了視覺化輸出模組的核心開發流程。

### `Ultrasonic_Radar.ino` (超音波測距與動態警示系統)
* **功能概述**：整合超音波感測器 (HC-SR04) 與蜂鳴器。程式會不斷測量前方障礙物的距離，並根據距離的遠近，動態調整蜂鳴器發聲的頻率（越近嗶聲越急促），模擬汽車倒車雷達的效果。
* **實作細節與硬體邏輯**：



| 核心技術與概念 | 說明與實作對應 |
| :--- | :--- |
| **超音波觸發時序 (Trigger Pulse)** | 根據感測器硬體規格，程式先將 `PIN_TRIG` 拉低 2 微秒確保訊號乾淨，接著送出一個 10 微秒的高電位脈衝 (`HIGH`) 來觸發超音波發射。 |
| **回波讀取與 `pulseIn`** | 發射後，`PIN_ECHO` 會轉為高電位，直到接收到反射回來的音波才降為低電位。程式利用 `pulseIn(PIN_ECHO, HIGH)` 精準計算這個高電位持續的時間（微秒，duration），即為音波來回的總時間。 |
| **聲速與距離換算 (Math Conversion)** | 聲音在空氣中的速度約為 340 m/s（每微秒 0.034 公分）。將總時間除以 2（單程時間）再乘上 0.034，即可簡化為程式中的除法公式：<br>👉 **公分 (cm)** = `duration / 58.0`<br>👉 **英吋 (inch)** = `duration / 148.0` |
| **動態警報節奏 (Dynamic Delay)** | 程式將測得的時間除以 100 轉換為延遲變數 `buzzDelay`。當障礙物越近，數值越小，`delay(buzzDelay)` 的時間就越短，造就了「距離越近、嗶聲越急促」的聽覺回饋。同時加入 `if (buzzDelay < 1) buzzDelay = 1;` 防止除數過小導致程式錯誤。 |

**總結**：
這個程式不僅展示了如何精準控制微控制器的 GPIO 時序來驅動感測器，更體現了「將輸入訊號 (Input) 經過數學運算後，連動影響輸出元件 (Output)」的核心控制邏輯，是嵌入式系統非常經典的整合演練。
